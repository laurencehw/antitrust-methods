# Antitrust Methods — 9-Hour Practitioner Short Course (South Africa)

A three-day (3 × 3-hour) course for competition-law practitioner masters
students, built from *Antitrust Methods: Research and Practice*. The course is
**South-Africa-forward** (the Competition Act, the Tribunal/CAC, and SA cases
are the spine; US/EU material is the comparison) and **concept-first** (R code
is shown as live demos, not required of students).

There are **two ways to get PowerPoint**, by design:

1. **Quarto → PowerPoint (primary).** `slides/day{1,2,3}.qmd` render to `.pptx`
   with the book's own toolchain, version-control as plain text, and can reuse
   the book's live R figures. This is the format you asked for.
2. **Prebuilt `.pptx` (fallback).** `slides/*_*.pptx` are ready to open and
   present right now — no Quarto/R needed. Use these if you want to start
   editing immediately or the Quarto render is inconvenient.

Both carry the same SA-forward, concept-first content.

## How the decks are built

### Option 1 — Quarto → PowerPoint (primary)

Render the `.qmd` decks to PowerPoint with the book's toolchain:

```bash
quarto render teaching/slides/day1.qmd --to pptx
# → teaching/slides/day1.pptx
```

The decks set `eval: false` by default, so they render to `.pptx` **without R
or any data** — every "code as demo" slide shows the R snippet and its expected
output as text. To turn the offline-safe demos into live figures (e.g.
`calc_hhi_change`, which needs no data or keys), set `eval: true` on that chunk
and render from the project root so `program/R/helpers.R` resolves. To brand the
deck, add a corporate template: `format: pptx: reference-doc: my-template.pptx`.

### Option 2 — prebuilt `.pptx` (fallback)

The `slides/*_*.pptx` files are generated by `deck_builder.py` (python-pptx) and
are ready to open and present:

```bash
cd teaching
pip install python-pptx        # one-time
python3 build_day1.py          # rewrites slides/day1_foundations.pptx
```

Or just open the `.pptx` in PowerPoint / Google Slides / Keynote and edit
directly. The palette matches the book (`program/R/helpers.R`): blue `#0072B2`
(US), orange `#E69F00` (EU), green `#009E73` (SA).

## Running the live R demos (optional)

Every demo points back to a function in `program/R/helpers.R`:

| Concept | Helper | Day |
|---|---|---|
| HHI / ΔHHI | `calc_hhi_change(shares, merging_firms)` | 1, 3 |
| Merger simulation | `run_logit_sim(products, merging_firms)` | 3 |
| Concentration/price series | `fetch_fred(series_id)` | 1 |
| Consistent figures | `theme_antitrust()`, `scale_colour_antitrust()` | all |
| Sensitivity tornado | `plot_tornado(sensitivity)` | 3 (litigation) |
| Decomposition / damages | `plot_waterfall(data)` | 2, 3 |

> **Caveat (see the teaching review).** As of this writing a fresh clone of the
> book cannot render every chunk (no committed `_freeze/` cache, and `data/`
> ships only a README). For live demos either (a) render the book once locally
> with API keys and commit `_freeze/`, or (b) rely on the decks, which show the
> expected output. The decks were built for option (b) so the class is never
> blocked by a render failure.

## Instructor notes — facts to confirm before teaching

The decks bake in the corrections from `REVIEW-2026-06-AUDIT.md`. The ones most
worth confirming against primary sources (your SA audience will know them):

- **Sasol** — the CAC **set aside** the excessive-pricing finding (2015). Teach
  as good law on the *test*, not standing liability.
- **Media24** — Tribunal finding (2015) **overturned by the CAC (2019)**.
- **Intel (EU)** — the rebates/AEC finding was **annulled** (GC 2022, CJEU Oct
  2024). Do not present "the AEC framework established in *Intel*" as settled.
- **Microsoft (US, 2001)** — the D.C. Circuit **vacated** the tying ruling;
  cite it for §2 monopoly *maintenance*.
- **Walmart/Massmart** — merger docket **73/LM/Nov10**; conditions, not a
  prohibition.
- **Burger King / Grand Parade** — prohibited (2021) purely on the
  spread-of-ownership limb of public interest, then approved with conditions.
- **SA dominance thresholds (§7)** — ≥45% presumed; 35–45% rebuttable; <35%
  only if power proven.
- **Labour cases** — *Jindal* and *DaVita* ended in **acquittals**; the FTC
  noncompete rule was set aside (*Ryan v. FTC*, 2024) and the appeal dropped
  (2025).

A South African practitioner read-through of the SA docket numbers and the
market-inquiry specifics (Grocery Retail, Healthcare, OIPMI) is still
recommended — this is an open item carried over from the manuscript audit.

## A note on the data exercises

Every [checkable] exercise has one correct number and a worked solution. They
are built on the book's own end-of-chapter "Data exercise (checkable)" items
(audit-verified) plus SA-flavoured extensions. Use them *during* the concept,
not only at the end — the arithmetic is short by design.
